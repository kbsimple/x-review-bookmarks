# Plan: Fix 8 Failing Tests

## Task
Fix 8 failing tests discovered during test suite audit. Tests broke after recent changes to stats command and Settings loading.

## Root Causes

### 1. StatsCommand tests (6 failures)
Tests mock `ReviewService` but not `PostsRepository`. Stats command now calls `posts_repo.get_post_stats()` (lines 1646-1647) which returns MagicMock, causing `'>' not supported between instances of 'MagicMock' and 'int'` error.

### 2. Settings tests (2 failures)
Tests expect `Settings()` to raise ValidationError when credentials missing, but `.env` file provides defaults. Tests need to clear env vars before testing.

Test asserts `tweepy.Client` called with `access_token=...` but code uses `bearer_token=...`. Tweepy uses `bearer_token` param for OAuth 2.0 access tokens (confusing naming). Test assertion is wrong - code is correct.
