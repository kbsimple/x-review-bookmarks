# Plan: Add Post Statistics to Stats Command

## Task
Add post statistics (oldest, newest, posts per month) to the `xbm stats` command, modeled after Claude's context usage display.

## Files

### src/repositories/posts.py
- **Action:** Add `get_post_stats()` method
- **Verify:** Returns dict with `oldest_date`, `newest_date`, `total`, and `by_month` data
- **Done:** Method queries posts table for aggregate statistics

### src/cli/main.py
- **Action:** Modify `stats` command to display post statistics in addition to review stats
- **Verify:** Output shows:
  - Post date range (oldest to newest)
  - Total posts
  - Posts per month breakdown (like Claude's model usage display)
- **Done:** Stats command outputs enhanced statistics

## Implementation Notes

1. `get_post_stats()` method should:
   - Get MIN(created_at) for oldest
   - Get MAX(created_at) for newest
   - Get COUNT(*) for total
   - Get COUNT per month via SQLite date functions

2. Display format inspired by Claude's context usage:
   - Show a visual bar or table for posts by month
   - Include date range header
   - Keep review stats below post stats
