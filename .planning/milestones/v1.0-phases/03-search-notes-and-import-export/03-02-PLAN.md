```python
def search_posts_combined(conn, query: str, author: Optional[str] = None, limit: int = 20):
    """Search posts with optional author filter.
    
    Supports:
    - Phrase search: "exact phrase"
    - Prefix search: term*
    - Boolean: term1 AND term2, term1 OR term2, -term3
    - Author filter: author:username
    """
    if author:
        fts_query = f"{{author_username author_display_name}} : {query}"
    else:
        fts_query = query
    
    cursor = conn.execute("""
        SELECT
            p.x_post_id,
            p.author_username,
            p.author_display_name,
            p.created_at,
            highlight(posts_fts, 0, '[match]', '[/match]') as highlighted_text,
            bm25(posts_fts) as rank
        FROM posts_fts
        JOIN posts p ON p.rowid = posts_fts.rowid
        WHERE posts_fts MATCH ?
        ORDER BY bm25(posts_fts)
        LIMIT ?
    """, (fts_query, limit))
    
    return [dict(row) for row in cursor.fetchall()]
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input to search query | Raw user input passed to FTS5 MATCH |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-04 | Tampering | Search query input | mitigate | _sanitize_query() escapes FTS5 special characters, use parameterized MATCH |
| T-3-05 | Denial of Service | Large search results | accept | Limit parameter prevents runaway queries; user controls limit |

## Security Controls

- `_sanitize_query()` removes/escapes FTS5 special characters: `(`, `)`, `*`, `-`, `'`, `"`
- Parameterized MATCH clause prevents SQL injection
- Limit parameter bounds result set size
</threat_model>
