# PLAN: Web Authentication

**Phase:** 6 (Web Foundation)
**Wave:** 2 of 4
**Plan:** 06-02
**Depends on:** 06-01 (FastAPI Foundation)
**Status:** ✅ Complete

## Goal

Enable web app to authenticate using shared CLI tokens from `data/tokens.json`.

## Requirements

- WEB-03: Web app authenticates using shared CLI tokens (data/tokens.json)

## Success Criteria

1. Web app loads tokens from `data/tokens.json` on startup
2. Unauthenticated users see "Run `xbm auth` first" message with instructions
3. Authenticated users can access all web routes
4. Token refresh handled automatically if expired
5. Session persists across requests without re-authenticating

## Implementation Plan

### Step 1: Create Auth Middleware

`src/web/auth.py`:
- `get_current_user()` dependency that loads tokens from `data/tokens.json`
- Returns `XAuth` object or raises `HTTPException(401)`
- Uses existing `load_tokens()` from `src/auth/oauth.py`

### Step 2: Auth Check on App Startup

`src/web/app.py`:
- Check for token file existence on startup
- Log warning if tokens not found (web still starts, but routes return 401)
- Don't block startup — allow running `xbm web` before `xbm auth`

### Step 3: Create Auth Status Route

`src/web/routes/auth.py`:
- `GET /api/auth/status` — returns `{"authenticated": true/false}`
- `GET /auth/login` — shows instructions page for unauthenticated users
- Login page displays "Run `xbm auth` to authenticate with X"

### Step 4: Session Handling (Optional)

For simplicity, tokens are read per-request from file. If needed later:
- Store tokens in server-side session (signed cookie)
- Refresh tokens automatically on 401 from X API
- For v1.1, simple file read is sufficient

### Step 5: User Context in Templates

`src/web/context.py`:
- `UserContext` dataclass with `username`, `authenticated` flag
- Pass to all templates via `request.state.user`
- Templates show login prompt or user info

### Step 6: Protected Routes

All routes that need auth use `Depends(get_current_user)`:
- Home page: public (shows posts if authenticated)
- Search: public (FTS5 works without auth)
- Settings: protected
- For v1.1, most routes are public since data is local-only

## Testing

1. Run `xbm web` without `xbm auth` — see login instructions
2. Run `xbm auth` then `xbm web` — authenticated
3. Check `/api/auth/status` — returns `{"authenticated": true}`
4. Delete `data/tokens.json` — back to unauthenticated state
