# Phase 14 — UI Design Contract: Static Export Viewer

> Visual and interaction contract for the generated `index.html` standalone bookmark viewer.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Context

This UI contract governs a **generated HTML artifact**, not a framework app. The executor writes
Python code (in `StaticExportService._write_index_html()`) that produces an `index.html` string
containing inline `<style>` and `<script>` blocks. The design contract below is the spec for what
that generated file must look and behave like.

**Viewer characteristics:**
- Personal-use only (the exporting user, not the public)
- 100–500 posts, all in memory
- Deployed to Netlify (HTTPS, fetch() loads JSON files)
- No external CDN links, no build step, no frameworks
- Primary use case: browse, search, filter, scan

**Shadcn gate:** Not applicable. Stack is vanilla HTML/CSS/JS (Python project). No React/Next.js/Vite.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tag pill internal padding (vertical), icon-text gap |
| sm | 8px | Tag pill internal padding (horizontal), meta row gap, badge gap |
| sm2 | 12px | Embedded/nested card internal padding and top margin (4×3 — compact nested spacing) |
| md | 16px | Card internal padding, search bar padding, control row spacing |
| lg | 24px | Gap between post cards in the list |
| xl | 32px | Header height padding, main content top/bottom padding |
| 2xl | 48px | Empty state vertical centering padding |
| 3xl | 64px | Page max-width side gutters on wide viewports |

Exceptions: sm2 (12px) — registered above. Multiple of 4 (4×3). Used exclusively for nested embedded card padding/margin to distinguish from outer card (md=16px).

## Typography

All sizes in px, rendered via inline `<style>`. System font stack (no web font fetch).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 15px | 400 | 1.6 | Post text, note text |
| Label | 13px | 400 | 1.4 | Author @handle, date, tag pills, topic pills, meta rows |
| Heading | 18px | 600 | 1.3 | Header title "X Bookmarks", section counts |
| Display | 13px | 600 | 1.4 | Review state badge label, "Quoting @username" label, "Reposted from @username" label |

Notes:
- Post text is NOT truncated in the static viewer (user locked CONTEXT.md: personal viewer,
  all posts in memory). Full text is rendered without ellipsis truncation.
- Quote tweet commentary text uses Body (15px/400).
- Embedded post text within nested card uses Body (15px/400) at 90% opacity.

## Color

Dark-mode-native palette. Consistent with the cast receiver (`receiver.html`) which uses #000
background + #fff text. The static viewer uses a slightly lighter dark (#0f172a slate-900
equivalent) to reduce eye strain on screens vs. TV.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #0f172a | Page background, main surface |
| Secondary (30%) | #1e293b | Post card background, header bar, footer |
| Accent (10%) | #2563eb | Tag pills background, "View on X" link, search/filter focus ring, post count badge |
| Embedded card surface | #111827 | Nested embedded-post card background (inside a secondary card) |
| Border | #334155 | Card border, nested card border, control dividers |
| Destructive | not applicable | No destructive actions in this viewer (read-only) |

Accent (#2563eb) reserved for:
1. Tag pill backgrounds
2. "View on X" external link text
3. Active/focus outline on search input and select controls
4. Post count badge background in header
5. Topic pill backgrounds (same blue, distinguishes from tag pills only by label prefix)

Text colors:
- Primary text: #f1f5f9 (slate-100) — post body, author display name
- Secondary text: #94a3b8 (slate-400) — @handle, date, meta labels
- Muted text: #64748b (slate-500) — "Reposted by", "Quoting", unavailable placeholder text
- Link text: #60a5fa (blue-400) — "View on X" on dark background (accessible contrast)

Rationale: dark palette chosen because (a) cast receiver already uses dark (#000), (b)
personal-use reading app benefits from lower eye strain in dark mode, (c) auto-mode design
principle "dark-mode-friendly" from objective.

## Layout

**Primary visual anchor:** The sticky header bar ("X Bookmarks" + post count badge) — this is the user's attention baseline on every scroll position. The post card list is the primary content zone; the header anchors context above it at all times.

### Page Structure

```
+--------------------------------------------------+
|  HEADER (sticky, secondary bg)                   |
|  "X Bookmarks"  [post count badge]               |
+--------------------------------------------------+
|  CONTROLS ROW (below header, secondary bg)        |
|  [Search input    ] [Date filter v] [Sort v]     |
+--------------------------------------------------+
|  MAIN (scrollable, dominant bg)                  |
|  [Post Card]                                     |
|  [Post Card]                                     |
|  ...                                             |
+--------------------------------------------------+
|  FOOTER (secondary bg, centered)                 |
|  "Exported YYYY-MM-DD · N posts"                 |
+--------------------------------------------------+
```

- Max content width: 720px, centered with `margin: 0 auto`
- Viewport: single-column layout only (personal viewer, no responsive grid needed)
- Header is `position: sticky; top: 0; z-index: 10`
- Controls row is NOT sticky (scrolls away with content)
- Post list is a vertical `<div>` with 24px gap between cards

### Post Card Anatomy

```
+------------------------------------------+
| CARD (secondary bg, 1px border, 8px radius) |
|  meta row: @author_username · YYYY-MM-DD  |
|  ── post type label (retweet/quote only) ── |
|  post body text (full, no truncation)     |
|  [media images grid if present]           |
|  ── embedded card (if retweet/quote) ──── |
|  |  @embedded_author · date             | |
|  |  embedded text                       | |
|  |  [embedded media if present]         | |
|  ── tag pills row ──────────────────── |
|  [#tag1] [#tag2]                         |
|  [Topic: topic name]                     |
|  review state row (if present)           |
|  "View on X" link (bottom right)         |
+------------------------------------------+
```

### Embedded / Nested Card

- Background: #111827 (embedded card surface)
- Border: 1px solid #334155
- Border-radius: 6px
- Padding: sm2 (12px)
- Margin-top: sm2 (12px) inside parent card
- Used for: quote tweet original, retweet original

### Media Grid

- 1 image: full-width, max-height 320px, object-fit: cover, border-radius 6px
- 2 images: CSS grid, 2 columns, gap 8px, each cell max-height 240px, object-fit: cover
- 3–4 images: CSS grid, 2x2, gap 8px, each cell max-height 200px, object-fit: cover
- Clicking an image opens it in a new browser tab (`target="_blank"`) — no lightbox
  (lightbox requires additional JS complexity; new tab is simpler for personal viewer)

## Controls

### Search Input

- Width: 100% minus controls for date + sort (flex layout)
- Placeholder text: "Search posts, authors, tags, topics..."
- Input type: `text`, `id="search-input"`
- Debounce: 150ms before triggering filter
- Clear behavior: empty input shows all posts (no X button needed)
- Styling: dark background (#0f172a), #f1f5f9 text, #2563eb focus ring (2px outline)
- Height: 40px (matches select controls)

### Date Filter Dropdown

- Element: `<select id="date-filter">`
- Width: 160px (fixed)
- Options (in order):
  - value="" label="All dates" (default, selected)
  - value="this_week" label="This week"
  - value="last_week" label="Last week"
  - value="this_month" label="This month"
  - value="last_month" label="Last month"
  - value="last_3_months" label="Last 3 months"
  - value="this_year" label="This year"
  - value="older" label="Older"
- Source: CONTEXT.md (locked)
- Triggers re-filter on `change` event (no debounce needed)

### Sort Dropdown

- Element: `<select id="sort-order">`
- Width: 160px (fixed)
- Options:
  - value="newest" label="Newest first" (default, selected)
  - value="oldest" label="Oldest first"
  - value="author" label="By author"
- Source: CONTEXT.md (locked)
- Triggers re-render on `change` event

### Result Count

- Location: header bar, right side of "X Bookmarks" title
- Format: "N posts" in accent-colored pill badge
- Updates on every filter/search change (reflects current filtered count)
- When filtered: shows "N of M posts"

## Component Inventory

These are the HTML component patterns the executor must implement as JS render functions.

| Component | JS Function | Notes |
|-----------|-------------|-------|
| Post card (original) | `renderOriginalCard(post, reviewState)` | Full text, media, tags, topics, review state, View on X |
| Post card (retweet) | `renderRetweetCard(post, reviewState)` | Attribution header + nested embedded card |
| Post card (quote) | `renderQuoteCard(post, reviewState)` | Commentary text + nested embedded card |
| Embedded card | `renderEmbeddedCard(embedded_post)` | Handles available=true and available=false |
| Unavailable placeholder | `renderUnavailablePlaceholder(embedded_post)` | SVG icon + "Original post unavailable" text |
| Tag pill | `renderTagPill(tagName)` | `#tagname` label, accent bg |
| Topic pill | `renderTopicPill(topicName)` | `Topic: name` label, accent bg, lower opacity than tag pills |
| Review state badge | `renderReviewBadge(reviewState)` | Shows next review date if present |
| Media grid | `renderMediaGrid(mediaUrls)` | 1/2/3-4 image layouts |
| Loading state | static HTML in page | Shown while fetch() promises resolve |
| Error state | `showError(message)` | Shown if fetch() fails |
| Empty state | `showEmptyState(reason)` | No posts in DB vs. no search results |

## States

### Loading State

- Shown immediately on page load while all 5 JSON files are being fetched
- Content: centered spinner (CSS animation, no external library) + "Loading bookmarks..."
- Spinner: 40px circle, 3px border, accent color top segment, 0.8s rotation
- Background: dominant (#0f172a)

### Empty State — No Posts Exported

- Triggered when: `posts.length === 0` after JSON load
- Heading: "No bookmarks found"
- Body: "Run `xbm sync` to fetch your bookmarks, then `xbm export-static` to regenerate this viewer."
- Icon: bookmark SVG (inline), 48px, muted color (#64748b)
- Layout: centered vertically with 48px top padding

### Empty State — No Search Results

- Triggered when: search/filter applied and filtered result count is 0
- Heading: "No posts match your search"
- Body: "Try different keywords or clear the date filter."
- No icon (keep it minimal)
- Result count badge shows "0 of N posts"

### Error State — JSON Load Failure

- Triggered when: any `fetch()` call fails (404, network error, CORS on file://)
- Heading: "Could not load bookmark data"
- Body: "Make sure you're viewing this file from Netlify, not by opening it directly. Direct file:// access does not support fetch()."
- Color: #ef4444 (red-500) for heading text
- Actionable: shows the failed URL if available

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Type in search box | Debounce 150ms, then re-filter and re-render post list |
| Change date filter | Immediate re-filter and re-render (no debounce) |
| Change sort order | Immediate re-sort and re-render |
| Click "View on X" | Opens X.com post URL in new tab (`target="_blank" rel="noopener noreferrer"`) |
| Click media image | Opens image URL in new tab (no lightbox) |
| Clear search box | Shows all posts for current date filter and sort |
| Combined filters | Search AND date filter applied simultaneously (both must match) |
| Page load | Show loading state → fetch 5 JSON files → build lookup maps → render all posts → hide loading |

## Implementation Notes for Executor

1. The entire `<style>` block is written as a Python triple-quoted string constant in
   `StaticExportService._write_index_html()`. CSS variables (`--color-bg`, `--color-card`,
   etc.) are recommended to keep the color palette maintainable within that string.

2. The `<script>` block uses vanilla ES6 (no transpilation). Use `const`, `let`, arrow functions,
   template literals, `Array.filter()`, `Array.sort()`, `Promise.all()`. No `var`.

3. Post rendering: build DOM strings with template literals and set `innerHTML` on the container.
   This is safe because all user content must be HTML-escaped before insertion. Use a helper:
   `function esc(s) { return String(s).replace(/&/g,'&amp;').replace(/</g,'&lt;').replace(/>/g,'&gt;').replace(/"/g,'&quot;'); }`

4. Tags and topics are stored in `posts.json` per-post. The JS does not need to load `tags.json`
   or `topics.json` for rendering — those files are used for potential future filtering by tag/topic.
   For this phase, render tags/topics from the per-post arrays in `posts.json`.

5. Review state is loaded from `review_state.json` and keyed by `post_id`. Build a Map:
   `const reviewMap = new Map(reviewData.review_states.map(r => [r.post_id, r]))`.
   Look up each post's state when rendering its card.

6. Date formatting: use `new Date(isoString).toLocaleDateString('en-US', {year:'numeric', month:'short', day:'numeric'})` for human-readable dates. Do not use a library.
