# Milestone v1.5: oEmbed Rich Embeds

**Status:** ✅ SHIPPED 2026-06-13
**Phases:** 15
**Total Plans:** 1

## Overview

Added `--rich-embeds` option to the `export-static` command that fetches native X/Twitter oEmbed HTML from the public oEmbed API at export time and stores it in `posts.json`. The static viewer renders oEmbed posts as native Twitter blockquote widgets via CDN-loaded `widgets.js`. Deleted/protected posts fall back to the existing custom card layout. The netlify-deploy skill was updated with a "deploy with rich embeds" trigger.

## Phases

### Phase 15: oEmbed Rich Embeds

**Goal:** Users can export the static viewer with native X widget rendering by passing `--rich-embeds` to export-static
**Depends on:** Phase 14 (Static Export)
**Plans:** 1 plan

Plans:
- [x] 15-01-PLAN.md — OEmbedService, StaticExportService rich_embeds param, CLI flag, viewer rendering, netlify-deploy skill update

**Details:**

Success Criteria:
1. `xbm export-static --rich-embeds` fetches oEmbed HTML from Twitter's public API for each post and stores it in posts.json as `oembed_html`
2. Deleted or protected posts get `oembed_html: null` and fall back to the existing custom card renderer
3. The static viewer renders posts with `oembed_html` as native Twitter blockquote widgets (Twitter widget JS loaded from CDN at view time)
4. Posts without `oembed_html` (--no-rich-embeds default) continue rendering with the existing custom card layout unchanged
5. The netlify-deploy skill is updated to accept an optional --rich-embeds flag and passes it to xbm export-static

## Milestone Summary

**Key Decisions:**
- `oembed_map` guard changed from `is not None` to truthy check: `if oembed_map:` — empty dict default path cleanly omits `oembed_html` field from all posts
- Patch target for lazy-imported OEmbedService is `src.services.oembed.OEmbedService` (definition site, not caller site) because of lazy import pattern
- OEMBED-03 (viewer JS rendering) is manual-only — JS is inline in a Python string with no JS test infrastructure in project
- `request_delay=0` in tests to skip `time.sleep` without needing to patch `time`
- Used `omit_script=true` in oEmbed API call to get bare blockquote HTML (no embedded script tag), then load widget.js once lazily

**Issues Resolved:**
- Bug: `oembed_map = {}` (empty dict) on default path passed `if oembed_map is not None:`, injecting `oembed_html: null` into every post even without `--rich-embeds`. Fixed to truthy guard.
- SKILL.md index table was missing "deploy with rich embeds" trigger (fixed during milestone audit)

**Issues Deferred:**
- 3 quick tasks acknowledged and deferred: limit-sync-flag, fix-failing-tests, stats-command (see STATE.md Deferred Items)

**Technical Debt Incurred:**
- OEMBED-03 viewer JS has no automated tests (inline JS in Python string, no JS test infra)
- `twttr.widgets.load(container)` on re-render is best-effort — Twitter SDK handles initial load; re-renders may not re-process newly visible embeds until SDK fully loaded
