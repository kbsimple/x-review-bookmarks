<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OEmbedService → publish.twitter.com | Outbound HTTP; response data treated as trusted HTML from Twitter's CDN and injected raw into the viewer via innerHTML |
| CLI user → xbm export-static | --rich-embeds is a boolean flag; no user-supplied URL or content reaches the oEmbed API |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Tampering | oembed_html in posts.json | accept | oembed_html originates from publish.twitter.com (Twitter's own CDN). Content is controlled by Twitter, not user input. Accept: no PII in embedded HTML, viewer runs in browser sandbox. |
| T-15-02 | Information Disclosure | oEmbed HTTP request | accept | Request contains only a public post URL (no auth, no PII). Connection is HTTPS. No credentials transmitted. Accept: low risk, standard public API usage. |
| T-15-03 | Denial of Service | OEmbedService.fetch_all rate limiting | mitigate | _DEFAULT_DELAY = 0.15s between requests prevents hammering. timeout=10.0 per request prevents indefinite hang. Network errors return None rather than raising. |
</threat_model>
