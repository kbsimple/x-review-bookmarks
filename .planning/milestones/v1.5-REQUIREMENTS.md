## v1.5 Requirements (Complete)

Requirements for Milestone v1.5 — oEmbed Rich Embeds.

### oEmbed (OEMBED)

- [x] **OEMBED-01**: `xbm export-static --rich-embeds` fetches and stores `oembed_html` in posts.json
  - Calls `https://publish.twitter.com/oembed?url=https://x.com/i/web/status/{id}&omit_script=true&dnt=true`
  - Returns blockquote HTML string per post
  - Stored as `oembed_html` field in each post object in posts.json
  - **Outcome:** Validated ✓ (Phase 15, commit 662f091)

- [x] **OEMBED-02**: Deleted/protected posts get `oembed_html: null`
  - HTTP 404 from oEmbed API → `fetch_oembed()` returns `None`
  - Network errors → `fetch_oembed()` returns `None`
  - `None` value stored as `null` in posts.json
  - **Outcome:** Validated ✓ (Phase 15, commit 662f091)

- [x] **OEMBED-03**: Viewer renders oEmbed HTML as native Twitter widget
  - Posts with `oembed_html` render via `renderOEmbedCard()` (blockquote injection)
  - `loadTwitterWidget()` loads `https://platform.twitter.com/widgets.js` once from CDN
  - Twitter widget SDK transforms blockquotes into rich embedded cards
  - **Outcome:** Validated ✓ manual — verified via Netlify deploy (Phase 15)

- [x] **OEMBED-04**: Posts without `oembed_html` (default path) render unchanged
  - `if oembed_map:` truthy guard prevents field injection when `oembed_map = {}`
  - Existing card renderers (original/retweet/quote) used when `oembed_html` absent
  - **Outcome:** Validated ✓ (Phase 15, commit 662f091; covered by test_no_rich_embeds_omits_oembed_html_field)
