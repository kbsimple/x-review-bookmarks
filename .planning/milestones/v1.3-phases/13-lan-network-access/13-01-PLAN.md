<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI → Localhost | User runs CLI locally, trusted |
| CLI → LAN network | User runs CLI, but LAN devices connect to server |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Spoofing | LAN certificates | mitigate | Use mkcert-generated CA certificates (not self-signed), require explicit `xbm lan-cert --generate` command |
| T-13-02 | Tampering | Network traffic | accept | HTTPS provides encryption; threat model is local trusted network |
| T-13-03 | Information Disclosure | LAN IP address | accept | LAN IP is only visible to local network; not a secret |
| T-13-04 | Denial of Service | Port binding | accept | User explicitly enables LAN access with --lan flag |
| T-13-05 | Elevation of Privilege | Certificate validation | mitigate | Block startup if certificates don't exist; don't fall back to self-signed |
</threat_model>
