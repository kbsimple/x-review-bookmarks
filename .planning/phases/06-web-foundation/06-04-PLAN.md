# PLAN: Search and Filter

**Phase:** 6 (Web Foundation)
**Wave:** 4 of 4
**Plan:** 06-04
**Depends on:** 06-03 (Post Browsing)
**Status:** ✅ Complete

## Goal

Implement full-text search and post filtering via web interface.

## Requirements

- WEB-05: User can search posts by text content (FTS5)
- WEB-06: User can filter posts by topic, author, and date range

## Success Criteria

1. Search bar in header returns FTS5 results
2. Topic filter shows posts matching selected topics
3. Author filter with autocomplete for known authors
4. Date range filter (from/to) narrows results
5. Filters combine with AND logic
6. Search and filter state visible in URL (bookmarkable)

## Implementation Plan

### Step 1: Create Search Routes

`src/web/routes/search.py`:
- `GET /search` — search page with results
- `GET /api/search` — JSON endpoint for HTMX search
- Query params: `q`, `topic`, `author`, `from_date`, `to_date`

### Step 2: Extend SearchService

`src/services/search.py`:
- Add `search_with_filters(query, topics, author, date_range)` method
- Combine FTS5 MATCH with WHERE clauses for filters
- Use prepared statements for SQL injection safety

### Step 3: Create Search Page Template

`src/web/templates/search.html`:
- Search bar (prominent, header position)
- Filter panel (collapsible on mobile)
  - Topic checkboxes (multi-select)
  - Author input (autocomplete)
  - Date range (two date inputs)
- Results grid (same as home page)

### Step 4: Implement Author Autocomplete

`src/web/routes/api.py`:
- `GET /api/authors?q={query}` — returns matching authors
- Query `posts` table for distinct `author_username` LIKE query
- Return JSON array: `["user1", "user2", ...]`
- Limit to 10 results

### Step 5: Topics API Endpoint

`src/web/routes/api.py`:
- `GET /api/topics` — returns all topics for filter dropdown
- Return JSON: `[{"id": 1, "name": "Python"}, ...]`

### Step 6: URL State Management

- Use query params for all filters: `/search?q=python&topic=ML&author=user123`
- HTMX replaces results without full page reload
- History API updates URL when filters change
- Shareable URLs for specific searches

### Step 7: Date Range Filter

`src/web/filters.py`:
- Parse date strings to `datetime` objects
- Handle edge cases: invalid dates, future dates
- Query: `WHERE created_at >= ? AND created_at <= ?`
