# PLAN: FastAPI Foundation

**Phase:** 6 (Web Foundation)
**Wave:** 1 of 4
**Plan:** 06-01
**Depends on:** Phase 5 (Spaced Repetition Resurfacing)
**Status:** ✅ Complete

## Goal

Establish FastAPI web application structure with HTTPS server and Jinja2 template engine.

## Requirements

- WEB-01: User can access application via web browser at localhost
- WEB-02: Web app serves posts over HTTPS (required for Google Cast)

## Success Criteria

1. FastAPI application runs on localhost with configurable port
2. HTTPS server serves content with self-signed certificate
3. Jinja2 templates render correctly with base layout
4. Health check endpoint returns 200 OK
5. Static files (CSS, JS, images) served correctly

## Implementation Plan

### Step 1: Add FastAPI and Dependencies

Add to `pyproject.toml` or `requirements.txt`:
- `fastapi` — web framework
- `uvicorn[standard]` — ASGI server with HTTPS support
- `jinja2` — template engine (bundled with FastAPI)
- `python-multipart` — form handling

### Step 2: Create Web Module Structure

```
src/web/
├── __init__.py
├── app.py          # FastAPI app factory
├── routes/
│   ├── __init__.py
│   └── home.py     # Home page route
├── templates/
│   └── base.html   # Base Jinja2 layout
└── static/
    └── .gitkeep
```

### Step 3: Implement FastAPI App Factory

`src/web/app.py`:
- Create FastAPI app with Jinja2 templates
- Mount static files directory
- Add health check endpoint at `/health`
- Configure template directory

### Step 4: HTTPS Certificate Generation

`src/web/certs.py`:
- Generate self-signed certificate for localhost
- Store in `data/localhost.crt` and `data/localhost.key`
- Auto-generate on first run if not present
- Use `cryptography` library for cert generation

### Step 5: Add Web Command to CLI

`src/cli/main.py`:
- Add `xbm web` command that starts the web server
- Accept `--host` and `--port` options (defaults: 127.0.0.1:8000)
- Load SSL context from generated certs
- Print URL for user to open in browser

### Step 6: Base Template

`src/web/templates/base.html`:
- HTML5 boilerplate with viewport meta
- Tailwind CSS CDN for styling (simple, no build step)
- Content block for page content
- Navigation placeholder

### Step 7: Settings Extension

`src/config/settings.py`:
- Add `web_host: str = "127.0.0.1"`
- Add `web_port: int = 8000`
- Add `cert_path: Path = Path("data/localhost.crt")`
- Add `key_path: Path = Path("data/localhost.key")`

## Testing

1. Run `xbm web` — server starts, prints URL
2. Open URL in browser — see home page (placeholder)
3. Verify HTTPS — browser shows cert warning, proceed works
4. Hit `/health` endpoint — returns `{"status": "healthy"}`
5. Check static files — serve a test CSS file
