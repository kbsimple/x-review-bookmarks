# PLAN: Custom Web Receiver

**Phase:** 7 (Cast Integration)
**Wave:** 2 of 4
**Plan:** 07-02
**Depends on:** 07-01 (Google Cast SDK Integration)

## Goal

Create a custom web receiver that displays post content on TV.

## Requirements

- RCVR-01: Custom Web Receiver displays post text and images on TV
- RCVR-02: Receiver handles post content loading from web app
- RCVR-03: Receiver displays post author and publication date

## Success Criteria

1. Receiver HTML displays post content in TV-friendly layout
2. Receiver receives post data via custom message channel
3. Large text readable from TV distance
4. Author username and date prominently displayed
5. Images scaled for TV aspect ratio

## Implementation Plan

### Step 1: Create Receiver HTML

- `src/web/templates/receiver.html` — Styled Media Receiver
- Full-screen layout for TV display
- Custom CSS for large text (3rem+)

### Step 2: Define Message Protocol

- Custom namespace: `urn:x-cast:com.bookmarked.posts`
- Message types: `LOAD_POST`, `NEXT_POST`, `PREV_POST`
- Post data structure: text, author, date, image_urls

### Step 3: Add Receiver Route

- `src/web/routes/cast.py` — Serve receiver page
- Route: `GET /receiver` — Returns receiver HTML
- Configure Cast app ID in settings

### Step 4: Style Receiver for TV

- Dark background for OLED TVs
- High contrast text (white on black)
- Large fonts (3-4rem for body text)
- Max-width for content centering
