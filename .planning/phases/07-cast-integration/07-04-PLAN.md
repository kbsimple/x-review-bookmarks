# PLAN: Mini Controller

**Phase:** 7 (Cast Integration)
**Wave:** 4 of 4
**Plan:** 07-04
**Depends on:** 07-03 (Cast Messaging)

## Goal

Add mini controller for navigation during active Cast session.

## Requirements

- CAST-04: Mini controller displays during active cast session
- CAST-05: Cast session state persists across navigation

## Success Criteria

1. Mini controller appears at bottom of screen when casting
2. Previous/Next buttons navigate through posts
3. Close button disconnects Cast session
4. Session state persists across page navigation
5. Current post info displayed in controller

## Implementation Plan

### Step 1: Create Mini Controller Component

- `src/web/templates/components/mini_controller.html` — Partial template
- Fixed position at bottom of viewport
- Shows: current post title, navigation buttons, disconnect button

### Step 2: Add Controller to Base Template

- Include mini controller in `base.html`
- Conditionally render based on Cast session state
- Use HTMX to show/hide based on session

### Step 3: Implement Navigation Queue

- `CastManager` maintains queue of posts to cast
- `nextPost()` and `previousPost()` methods
- Track current position in queue

### Step 4: Persist Session Across Pages

- Store session reference in `window.CastContext`
- Re-attach to existing session on page load
- Update mini controller state on navigation
