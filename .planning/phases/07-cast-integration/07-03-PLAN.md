# PLAN: Cast Messaging

**Phase:** 7 (Cast Integration)
**Wave:** 3 of 4
**Plan:** 07-03
**Depends on:** 07-02 (Custom Web Receiver)

## Goal

Implement message passing between sender (web app) and receiver (TV).

## Requirements

- CAST-03: User can cast post content to TV screen
- RCVR-02: Receiver handles post content loading from web app

## Success Criteria

1. Clicking a post card casts it to TV
2. Receiver receives post data and displays it
3. Sender receives acknowledgment from receiver
4. Error handling for failed message delivery
5. Loading states while post loads on TV

## Implementation Plan

### Step 1: Extend CastManager

- Add `loadPost(postId)` method to `CastManager`
- Fetch post data from `/api/posts/{postId}`
- Send `LOAD_POST` message via Cast session

### Step 2: Update Browse Template

- Add `cast-post` button to post cards
- Only show when Cast session is active
- Wire click handler to `CastManager.loadPost()`

### Step 3: Implement Receiver Message Handler

- Listen for `LOAD_POST` messages in receiver
- Parse post data and update DOM
- Send acknowledgment back to sender

### Step 4: Add Loading States

- Show "Casting..." spinner on sender
- Receiver shows loading animation while parsing
- Fade transition when content updates
