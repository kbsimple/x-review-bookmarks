# PLAN: Google Cast SDK Integration

**Phase:** 7 (Cast Integration)
**Wave:** 1 of 4
**Plan:** 07-01
**Depends on:** Phase 6 (Web Foundation)
**Status:** In Progress

## Goal

Integrate Google Cast SDK into the web app for device discovery and session management.

## Requirements

- CAST-01: User sees Cast button in web app header when devices are available
- CAST-02: User can connect to Chromecast/Smart TV devices from web app

## Success Criteria

1. Cast button appears in navigation when Chromecast devices detected
2. Clicking Cast button shows device selection popup
3. Selecting a device establishes Cast session
4. Cast state (connected/disconnected) tracked and displayed
5. Session disconnect handled gracefully

## Implementation Plan

### Step 1: Add Cast SDK Script

- Add Google Cast SDK script to `base.html`
- Initialize Cast API with app ID (use default media receiver ID initially)
- Request session with `chrome.cast.requestSession()`

### Step 2: Create Cast Manager Module

- `src/web/static/js/cast.js` — Cast sender initialization
- `CastManager` class handles session lifecycle
- Event listeners for session state changes

### Step 3: Add Cast Button to Navigation

- Update `base.html` with Cast button in nav
- Button shows when devices available, hidden otherwise
- Use Cast icon (SVG) from Google Material Icons

### Step 4: Session State Management

- Track active session in `CastManager`
- Expose session state to other components
- Handle disconnect/reconnect scenarios
