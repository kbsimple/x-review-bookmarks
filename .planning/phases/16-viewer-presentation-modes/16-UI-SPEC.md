## Interaction Contract

### Mode Switching

| Trigger | Behavior |
|---------|----------|
| Click `[Stream]` button | `setMode('stream')` — restore `savedScrollY`, re-render as stream |
| Click `[Carousel]` button | `setMode('carousel')` — save current `window.scrollY`, render carousel at index 0 |
| `Escape` key (when `currentMode === 'carousel'`) | `setMode('stream')` |
| Page load | Read `localStorage.getItem('xbm_mode')` — default to `'stream'` if null |

### Carousel Navigation

| Trigger | Behavior |
|---------|----------|
| Click `← Prev` button | `carouselIndex--; renderView(); window.scrollTo(0, 0)` |
| Click `Next →` button | `carouselIndex++; renderView(); window.scrollTo(0, 0)` |
| `ArrowLeft` key | Same as `← Prev` click — no-op if at index 0 |
| `ArrowRight` key | Same as `Next →` click — no-op if at last index |
| Prev button at index 0 | Button rendered with `disabled` attribute + `opacity: 0.3; cursor: not-allowed; pointer-events: none` |
| Next button at last index | Button rendered with `disabled` attribute + `opacity: 0.3; cursor: not-allowed; pointer-events: none` |
| Filter/search/sort change | `carouselIndex` resets to 0 inside `renderView()` at start of carousel branch |
| No wrap-around | Confirmed: no circular navigation |

### Keyboard Listener Contract

- Single `document.addEventListener('keydown', ...)` added once at page load in the existing event listener block (lines 860–865).
- Fires only when `currentMode === 'carousel'` — early return otherwise.
- Keys handled: `ArrowRight`, `ArrowLeft`, `Escape`.
- `Up` / `Down` arrows pass through (page scroll, browser default).
- Input fields: keyboard listener fires globally, but does not intercept when focus is on `#search-input` — exclude with `if (document.activeElement === document.getElementById('search-input')) return;`

### State Persistence

| State | Storage | Key | Values | Reset Condition |
|-------|---------|-----|--------|----------------|
| Active mode | `localStorage` | `xbm_mode` | `'stream'` / `'carousel'` | Never auto-reset |
| Carousel index | JS `let carouselIndex` | — | `0..N-1` | Reset to 0 on any `renderView()` call in carousel mode |
| Stream scroll Y | JS `let savedScrollY` | — | `0..Infinity` | Overwritten on each switch to carousel |

## oEmbed Integration Contract

oEmbed posts must render correctly inside the carousel. The following sequence is mandatory inside `renderCarousel()`, called after `innerHTML` is assigned:

```javascript
if (post.oembed_html) {
  loadTwitterWidget();                          // guarded by _twitterWidgetLoaded flag
  if (window.twttr && window.twttr.widgets) {
    window.twttr.widgets.load(               // scope to #post-list, not full document
      document.getElementById('post-list')
    );
  }
}
```

Source: RESEARCH.md verified pattern, mirrors stream path at static_export.py lines 851–856.

## Anti-Patterns (Executor Guard)

These are explicitly prohibited by RESEARCH.md findings:

| Pattern | Why Prohibited | Correct Approach |
|---------|---------------|-----------------|
| `document.addEventListener('keydown', ...)` inside `renderView()` | Accumulates N listeners after N renders — carousel skips multiple posts per key | Wire once at page load |
| `filterAndSort()` called inside `renderCarousel()` | Double-invocation per render | Pass `results` as parameter from `renderView()` |
| `twttr.widgets.load(document)` | Scoped to full DOM; use container element | `twttr.widgets.load(document.getElementById('post-list'))` |
| `carouselIndex` not reset on filter change | Out-of-bounds access, `renderPost(undefined)` throws | Reset at top of carousel branch in `renderView()` |
| Unescaped strings in new `innerHTML` | XSS via user bookmark content | Use existing `esc()` helper for all dynamic content |
| Transition/animation on mode switch | Adds complexity with no functional benefit | No animation — direct DOM swap |
