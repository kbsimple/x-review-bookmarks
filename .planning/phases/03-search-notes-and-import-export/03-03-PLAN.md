       ```python
           def import_json(
               self,
               input_path: Path,
               conflict: str = "skip"  # "skip" or "update"
           ) -> ImportResult:
               """Import posts from JSON export file.
               
               D-03: Import supports JSON format only.

                   input_path: Path to JSON export file
                   conflict: "skip" to skip existing posts, "update" to update them
               
               Returns:
                   ImportResult with imported and skipped counts
               
               Raises:
                   ValueError: If version or source fields are invalid
                   FileNotFoundError: If input file doesn't exist
                   json.JSONDecodeError: If file is not valid JSON
               """
       ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Import file path | User-provided path to JSON file |
| Import file content | User-provided JSON data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-06 | Tampering | Import file path | mitigate | Validate path exists, is a file, is within expected directory |
| T-3-07 | Tampering | Import file content | mitigate | Validate version and source fields before processing |
| T-3-08 | Denial of Service | Large import file | accept | User controls import file; no automatic import |
| T-3-09 | Information Disclosure | Export file content | accept | Export is intentional user action; file contains user's own data |

## Security Controls

- Version validation: Reject import files with version != "1.0"
- Source validation: Reject import files with source != "xbm"
- Path validation: Check file exists before attempting to read
- Error handling: Capture errors without crashing, return in ImportResult.errors
</threat_model>
